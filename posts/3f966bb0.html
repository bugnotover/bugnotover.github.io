系统运维技术-PXE自动化安装系统

系统运维
系统运维
发布日期:   2021-07-05
更新日期:   2021-07-05
文章字数:   1.2k
阅读时长:   5 分

实验环境：

  1. 一台 Win10_x64 物理主机
  2. 安装 Vmware workstation15 pro 虚拟机平台
  3. 物理机安装 finalshell 运维工具
  4. VM 中安装 CentOS-7-x86_64 系统

PXE 安装优点，这种安装系统的方式可以不受光驱，光盘以及一些外部设备的限制，还可以做到无人值守，大大减轻了运维人员的工作负荷，像在一些主机数量庞大的机房进行批量安装，PXE 将是很好的选择。

一、建立本地 yum 源

1.1 yum 的配置文件

位置在/etc/yum.repos.d/*.repo

1.2 挂载光盘

mkdir  -p  /mnt/cdrom
mount /dev/cdrom  /mnt/cdrom

1.3 挂载本地源

备份 yum 配置文件，重新建立/etc/yum.repos.d/文件夹

mv /etc/yum.repos.d/  /etc/yum.repos.d.bak

mkdir /etc/yum.repos.d/

创建本地 yum 文件命令：

yum-config-manager --add-repo=file:///mnt/cdrom

修改配置文件，最后一行添加 gpgcheck=0

vi /etc/yum.repos.d/mnt_cdrom.repo

列出配置文件内容 #cat /etc/yum.repos.d/mnt_cdrom.repo

更新 cache 使用本地 yum 源

yum clean all && yum makecache

二、安装软件包

2.1.需要安装的包主要包括：vsftp tftp dhcp syslinux

配置文件包括 tftp dhcp default ks.cfg

yum install system-config-kickstart –y
yum install dhcp tftp tftp-server syslinux vsftpd xinetd
yum install syslinux  -y

三、设置配置项、拷贝 tftp 引导文件

3.1. 设置 dhcp 其中 dhcp 配置文件修改如下：

vi /etc/dhcp/dhcpd.conf

–

subnet 192.168.157.0 netmask 255.255.255.0 {range 192.168.157.133 192.168.157.253;
option domain-name-servers 192.168.157.134;
option routers  192.168.157.2;
option broadcast-address 192.168.157.2;
default-lease-time 6000;max-lease-time 7200;
# IP of PXE Servernext-server 192.168.157.134;filename "pxelinux.0";}

3.2 配置 tftp

vim /etc/xinetd.d/tftp

service tftp disable 值改为 no，唤醒该进程。

3.3.  TFTP 拷贝引导文件操作

cp -v /usr/share/syslinux/pxelinux.0 /var/lib/tftpboot
cp -v /usr/share/syslinux/menu.c32 /var/lib/tftpboot
cp -v /usr/share/syslinux/memdisk /var/lib/tftpboot
cp -v /usr/share/syslinux/mboot.c32 /var/lib/tftpboot
cp -v /usr/share/syslinux/chain.c32 /var/lib/tftpboot
mkdir /var/lib/tftpboot/pxelinux.cfg
mkdir /var/lib/tftpboot/centos7
cp /mnt/cdrom/isolinux//vmlinuz /var/lib/tftpboot/centos7/
cp /mnt/cdrom/isolinux/initrd.img /var/lib/tftpboot/centos7/

四、创建 kickStart & PXE menu file.

4.1 创建 root 密码加密字符串

openssl passwd -1 yjb9814.

4.2 创建 kickstart 配置文件

vi /var/ftp/pub/centos7.cfg

4.3 创建 PXE 菜单文件

vi /var/lib/tftpboot/pxelinux.cfg/default
  • 易错：ks 文件一定要注意路径正确，否则会报错 failed to fetch kickstat from ftp

出现这个问题，也可能是 selinux 没关或 ftp 未设置匿名模式。

五、拷贝光盘文件到/var/ftp

mkdir  /var/ftp/centos7

cp -a /mnt/cdrom/\*  /var/ftp/centos7

六、开启和自启动 xinetd, dhcp and vsftpd service.**

systemctl start xinetd
systemctl enable xinetd
systemctl start dhcpd.service
systemctl enable dhcpd.service
systemctl start vsftpd
systemctl enable vsftpd

出现错误，查看 dhcp 状态

配置主机网卡 ip 地址与配置文件中地址在同一地址段，

也就是为虚拟机系统配置静态 ip

使用命令 service network restart 重启网卡，然后重新设置自启动，解决问题。

七、关闭防火墙与 Selinux

7.1.关闭防火墙

临时关闭  

systemctl stop firewalld

查看防火墙状态

systemctl status firewalld

永久关闭防火墙  

systemctl disable firewalld

如果在关闭防火墙过程中出现下面的错误：

使用命令，移除防火墙再继续操作。

yun remove firewalld

7.2.关闭 Selinux

查看 selinux 状态#

getenforce

enforcing         表示启动

临时关闭    

setenforce  0

永久关闭（修改配置文件，即可永久关闭）

vi /etc/selinux/config

SELINUX=enforcing    修改为"SELINUX=disabled"

八、测试 ftp 通信方式自动安装

新建一台虚拟机，为节约时间成本，可以移除非必要设备。

如果在启动过程中出现报错可以参照

https://wenku.baidu.com/view/6dcea6290066f5335a812119.html

文章中所说进行解决。我在安装时出现了

failed to fetch kickstat from ftp

经过修改/var/lib/tftpboot/pxelinux.cfg/default 文件中 ks 路径，系统找到了 ks 文件路径，完成了自动化安装。

九、添加 http 安装

9.1.挂载本地源和光盘、拷贝镜像

mkdir -p /var/www/html/centos7.8
mkdir -p /var/www/html/ks
Mount /dev/cdrom  /mnt/cdrom
cp -a /mnt/cdrom/\*  /var/www/html/centos7.8

9.2.配置 tftp 引导文件

vi /var/lib/tftpboot/pxelinux.cfg/default

添加 http 安装方式

menu label ^Install CentOS 7.8
menu default
kernel /centos7.8/vmlinuz
APPEND initrd=/centos7.8/initrd.img  ks=http://192.168.157.134/ks/centos7.8.cfg

挂载安装文件

mount  /dev/cdrom
mount /dev/cdrom  /mnt/cdromu

9.3．复制配置文件

cp /mnt/cdrom/isolinux/vmlinuz /var/lib/tftpboot/centos7.8/

cp/mnt/cdrom/isolinux/initrd.img/var/lib/tftpboot/centos7.8

9.4.配置 ks 文件

[root@localhost ks\]# vi  centos7.8.cfg

配置完成后即可进行新建虚拟机，可以看到增加了 http 安装方式
